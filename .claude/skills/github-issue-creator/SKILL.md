# GitHub Issue Creator

## Before writing

Read `AGENTS.md`, `docs/00_Core/SESSION_LIFECYCLE.md` (LOAD), `docs/10_Development/10_Agent_Protocol.md`, and vault `00_System/invariants/_index.md` (or `Architecture Invariants.md` entry).

## Issue body checklist

- Problem statement and motivation
- Acceptance criteria (testable)
- Out of scope
- Constraints (security, performance, data paths)
- Links to related PRs/Issues

## Labels and metadata

Suggest labels (type, priority, area) consistent with team practice.

## Branch hint

Include a suggested branch name matching `AGENTS.md` conventions.
